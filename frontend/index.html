Présentation du projet FullStack3Tiers

Ce projet est une application full-stack-3Tries qui inclut des services pour le frontend, le backend, la base de données, le monitoring, et le reverse proxy avec Traefik. j'ai mis en place un environnement de préproduction avec Docker et un environnement de production avec Kubernetes, tout en automatisant le déploiement avec GitHub Actions

Problémes rencontrés et Résolution

Probléme 1 : "Aucun fichier ou répertoire nommé kubectl"

Le fichier kubectl n'existe pas, cela peut étre un probléme dans le téléchargement.

Solution : Télécharger le fichier é nouveau et répétez les étapes.

Probléme 2 : "Erreur d'intégrité lors de la vérification du fichier"

Si la commande de vérification d'intégrité échoue, cela peut signifier que le fichier téléchargé est corrompu ou qu'il ne correspond pas é la somme de contréle attendue.

Solution : Réessayez le téléchargement ou vérifiez que l'URL du fichier SHA256 est correcte.

  • Curl : Un utilitaire de ligne de commande utilisé pour transférer des données via des protocoles réseau.
  • Sha256sum : Un outil utilisé pour générer ou vérifier des sommes de contréle (hash) en utilisant l'algorithme SHA-256. Cela garantit que le fichier téléchargé est exact et n'a pas été altéré.
  • Chmod : Une commande Linux utilisée pour modifier les permissions des fichiers. Nous l'utilisons pour rendre le fichier kubectl exécutable.

Conclusion

En conclusion, le projet FullStack3Tiers est un excellent moyen de mettre en pratique les concepts de déploiement d'applications full-stack avec Docker et Kubernetes. Il permet de comprendre comment les différents services interagissent les uns avec les autres et comment les automatiser pour un déploiement plus efficace.
